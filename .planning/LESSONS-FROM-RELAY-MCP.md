# Lessons from relay-mcp (the abandoned predecessor)

**Compiled:** 2026-06-09 from an exhaustive 18-agent read of `/Users/ghanavati/ai-stack/Projects/relay-mcp` (81,358 lines src, 344 files, 4,787 commits, ~190 session logs, full docs/planning). No inference — every claim traces to a file the agents read.

## What relay-mcp was, and how it died

A model-agnostic AI delegation + audit layer (engine ~30% of the code) that grew an enterprise governance/compliance platform around itself (~70%). Its own 2026-04-05 redesign said: *"~14,500 lines... 70% is enterprise governance infrastructure built for a regulated enterprise buyer that doesn't exist yet."* It then **grew to 81K lines** chasing finance MRM → medical-device → SupTech verticals, a hosted React control plane, a skill marketplace, and a self-rewriting loop. The 2026-05-03 sessions are its death certificate: *"moving away from relay-mcp to relay as a solo cli product, this relay-mcp folder being sunsetted."* **The successor it names is THIS current `Relay/` project — already lean (2 deps, no MCP SDK), having dropped ~60% of relay-mcp scope.** "We've been here before" = bloat → sunset → lean rebuild. Phase 9 must not re-import the bloat.

## By the numbers (the autopsy)

- **4,787 commits in ~10 weeks; 2,509 (52.4%) were `docs(session)` spam** (many byte-identical), real engineering ~22.5%. `docs/sessions/` = 183 files / 26 MB.
- **The product `.relay/relay.db` is 0 bytes — zero runs ever recorded** — under a README whose entire pitch is *"Every AI action traced, diffed, and queryable. Backed by SQLite."* It also sold *"Relay Enterprise... $50-500K/yr ACV"* and *"19 report formats (SR 11-7, EU AI Act, DORA...)"* while STATE claimed "v1.3 complete 26/26" and BUGS.md carried **96 open bug markers**.
- **Hosted = a full multi-tenant SaaS** (Postgres + OAuth GitHub/Google/Okta + AES-GCM credential vault + billing no-op stub + 11-screen React dashboard) built over ~5 weeks for one undocumented env flag; its central `StoreAdapter` (186 lines) was **never wired to the runtime** — and the *last* hosted commit was a coupling-CUT, Stripe was added-then-deleted. The project was already reversing the SaaS bet before it died.

## The failure modes (what killed it) — do not repeat

1. **Vertical scope capture.** 44% of the CLI (5,492 lines, 15 `report-*.ts`) was jurisdiction regtech (EIOPA/DORA/SR 11-7/EU AI Act). 45 MCP tools, ~30 were SR-11-7/MRM compliance CRUD. The finance/MRM pivot was *"compiled from a Grok conversation"* — chatbot-seeded, not user-need. The fact-check flagged its own market stats as *"likely fabricated"* (`63% non-developers`, `$4.7B market`, `Moltbook 1.5M keys`). → **Stay a focused agnostic memory+control layer. No verticals. No positioning on unverified numbers.**
2. **Assess-not-ship.** *"Assess → Document → Scaffold → Stop. The scaffold exists. The wire does not... The scaffold is more dangerous than nothing."* `IdempotencyStore`, jobs state machine, `MemoryStore.lint()`, `compound-aggregation` — all built, tested, **orphaned**. → **Every store/module task ends with a wiring + integration-test task. Behavioral acceptance criteria, never grep/wc metrics.**
3. **No single run-lifecycle owner → data loss.** *"Three parallel persistence systems caused data loss."* SQLite was *"added alongside the old JSONL/checkpoint system instead of replacing it."* `RunHandle.complete()` became a god-method doing persist+intent+cost+review+drift+guard+memory+retention best-effort in one path. → **One injected `RunLifecycle`/persistence interface from day 1. Durable staged records, not one best-effort path.**
4. **Fail-open verification (BUG-15, CRITICAL).** Default reviewer hardcoded to a 404'ing model; *"89 verifications, zero genuine reviews... entire v1.2 verification history tainted,"* surfaced only on stderr. Overcorrected later to record absence as `rejected`. → **Explicit `unavailable` state, surfaced in the run record/default output. Never fail-open, never fake-reject.**
5. **Multi-session context bleed (BUG-30).** `run_history` injected runs from ALL sessions into every worker; *"no authoritative machine-readable source of which session owns what."* `onyx-auto-attach.ts` deleted 3× because parallel sessions had no record it was intentional. → **Session ownership is first-class and machine-readable, not bolted on.**
6. **Two sources of truth for "provider."** A genuinely agnostic env-config registry (`config/providers.ts`, `registry.ts`) coexisted with a hardcoded `RelayProviderId` union + `agent-catalog.ts` + `effort-router.ts` inlined model IDs (and a phantom `gemini` in `agent-factory.ts`, imported nowhere). → **Provider/model identity is DATA (config/registry), never a TS union or inlined string. One registry.**
7. **Duplicate DB-path literal.** `~/.relay-mcp/relay.db` (runtime) vs `.relay/relay.db` (8 report CLIs) → the 0-byte db is a report run opening the wrong path. → **One `resolveDbPath()` imported everywhere.**
8. **MCP tool sprawl + hand-synced lists.** 44 tools (host pays context for all of them); `SERVER_TOOL_NAMES` a manual array that drifted (BUG-19). MCP was the *only* transport (GAP-13, "too narrow"). → **Single-digit core tool set; derive the tool list from the registry; MCP is ONE transport, CLI stays primary.**
9. **Doc ≠ reality, everywhere.** `docs/schemas/` never wired; SECURITY.md "redaction implemented" vs data-retention "redaction planned"; self-reported "FIXED" bugs that weren't (BUG-38); stale test counts; `AGENTS-COMPACT.md` auto-dup. *"Existence ≠ Implementation."* → **Plan against live `src:line` grep, not docs. Ship a SECURITY.md claim only when the code path is wired + tested.**
10. **Premature hosted/UI + no migration retirement.** Hosted React control plane (backend stubbed) before a 2nd user. New systems added beside old ones, never replacing. → **Local-first; no hosted/UI until real demand; every new store has a deletion plan for what it replaces.**
11. **Cost-aware was policy-only.** `GAP-13`: routing/RTK "policy-only, not mechanically enforced." Cost was OpenRouter `usage.cost` passthrough — no price table, no cost-based routing for other providers. → **Real per-model price map; cost computed from tokens for every provider; legible routing (`operations/routing-policy.md`: "cheapest lane that works; do not hide routing choices").**
12. **Process scar tissue.** Session-log commit spam; a doc-sync post-commit hook that auto-reverted hand-written STATE fields; long "recurrent failure" prose rules nobody obeyed. → **Enforce via hooks/tests, not prose. Don't auto-overwrite state. Don't commit session-log noise.**

## What was genuinely good — REUSE (port, don't reinvent)

- **The agnostic provider layer already existed.** `config/providers.ts` env-discovery (`RELAY_PROVIDER_<NAME>_URL|TYPE|KEY|HEADER_*|ADAPTER`) + `GenericHttpRunner` as default + thin native subclasses (Anthropic, Codex-subprocess) + `ProtocolAdapter` (anthropic / openai-chat / openai-responses) in `local-tools.ts`. **This IS the "talk to any model" the user wants — port it, delete the closed union.**
- **Engine:** `delegate` / `delegate_parallel` (dependency-wave scheduler + worktree isolation) / `run-recorder` / `run-store` (optimistic `version` column) / append-only `run_events` + immutable records via triggers.
- **Memory:** SQLite + FTS5, **no embeddings/vectors** (model-agnostic, zero deps); wiki-upsert by `entity_key`; token-budgeted greedy recall + per-type decay half-lives; trust tiers derived from `success_recall_count`; `[UNVERIFIED]`/`⚠ FAILED` injection prefixes; opt-in layered context with a read-time injection guard. (Roadmap idea worth taking: hierarchical scoped memory global→team→workspace→run instead of one flat table.)
- **Control model:** operator/worker promotion ladder — *"Workers do work. Operators promote truth"* — transcript → provisional → verified → promoted, every promotion reversible; 4 model-agnostic JSON schemas (uuid + `state_revision` + `lifecycle.valid_transitions` + `reversal`); bounded handoff packets ("continuity depends on bounded loading, not maximal").
- **Security:** `redaction.ts` (stateless), `middleware.ts` (SSRF + workdir denylist + realpath, mutex-first validation), concurrency mutex-with-drain, single `~/.relay/.env` loaded by the tool itself (the real model-agnostic secret mechanism), `RELAY_MCP_VARIANT=dangerous` read-only-by-default gating, strict stdout discipline.
- **MCP primitive:** `McpServer.registerTool(name,{inputSchema: zod},handler)` + one `toMcpResult` envelope — correct, just keep it LEAN and derive the tool list.
- **Cost:** `cost_events`, `context_mode: full|minimal` (drops ~10K tokens/dispatch), `INPUT_TOKEN_CAP` preflight, the routing-policy stance. MCP-vs-CLI token finding confirmed: *"~30× token difference per operation"* — validates CLI-primary + MCP-optional, cost-aware.

## How this reshapes the MCP plan (Phase 9)

The Phase 9 plans (`.planning/phases/09-mcp-server/`) are sound for the **MCP protocol surface**, but the predecessor proves the real work and the real risk are elsewhere:

- **Sequence flips.** "Talk to any model" (port the env-config provider registry + GenericHttpRunner + ProtocolAdapter; delete any closed provider list) is the foundation and must come BEFORE the MCP dispatch surface. MCP is a thin lean cap on top, not the center.
- **Hard guardrails to write into the plan:** single-digit MCP tool count; tool list derived from the registry (no hand-synced array); `resolveDbPath()` single helper; one `RunLifecycle` owner; provider identity as config data only; cost = real price-table computed per provider, routing legible; verification/any-fallback surfaces an explicit `unavailable` in the record; session ownership machine-readable; every store task ends with a wiring + integration test (matches the PAUL audit finding already applied to 09-06).
- **Explicitly out of scope (the bloat that killed v1):** compliance/regtech tools, hosted/UI, skill marketplace, self-improve loop, MCP-as-only-transport, any vertical.
