<objective>
Add and pin the official Model Context Protocol TypeScript SDK as a new runtime dependency, then build a small build-time probe that proves the installed package's import surface (`McpServer`, `StdioServerTransport`) is the one we will import from — verified against the package on disk, never hardcoded from research docs.

Purpose: MCP-07 demands the exact package name and import paths be verified against the installed version, because the SDK has shifted between `@modelcontextprotocol/sdk` (with `/server/mcp.js` + `/server/stdio.js` subpaths) and a newer `@modelcontextprotocol/server` style. A new dependency also triggers the project's supply-chain gate (verify name + version + maintainer before install). This plan isolates that risk so later plans build against a known-good surface.

Output: a pinned SDK dependency in package.json/package-lock.json, and `src/mcp/sdk-probe.ts` exporting `resolveMcpSdk()` that the server (Plan 05) calls before connecting.
</objective>
