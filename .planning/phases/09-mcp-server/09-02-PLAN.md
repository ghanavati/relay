<objective>
Establish the two foundations every MCP tool wrapper needs: (1) the existing control-tool Zod schemas become exported, named single-source-of-truth definitions, and (2) a small `src/mcp/result.ts` that owns the MCP envelope, boundary redaction, and RelayError→MCP-error mapping in one place.

Purpose: MCP-05 requires MCP tool input schemas to BE the existing Zod boundary schemas, not a hand-maintained JSON Schema that drifts. Today `src/control/tools.ts` keeps its Zod schemas (`ListToolArgsSchema`, `SendToolArgsSchema`, …) module-private and exposes only OpenAI-style `ToolDef.parameters` JSON. The MCP SDK's `registerTool` wants a Zod object as `inputSchema`, so we must export those schemas and prove both paths share one definition. MCP-06 requires RelayError to map to an MCP tool-error result and `REDACTION_PATTERNS` to run before any value crosses the boundary; centralizing that in result.ts keeps every tool wrapper honest and prevents a leaky path.

Output: exported Zod schemas + a name→schema map in src/control/tools.ts (additive — agentic path unchanged), and src/mcp/result.ts with `toMcpResult` / `relayErrorToMcpResult` / `withMcpResult`.

Note: the memory schemas (`recallSchema` / `RecallArgsSchema`, `rememberSchema` / `RememberArgsSchema`) are ALREADY exported from src/contracts/memory.ts — no change needed there. This plan only closes the control-tool side of the single-source-of-truth requirement.
</objective>
