<objective>
Lay the two SDK-independent foundations of the MCP server: (1) the pinned, build-time-verified MCP SDK dependency with a probe that proves the import surface against the installed package, and (2) `src/mcp/result.ts` — the one module that owns the MCP envelope, boundary redaction, and RelayError→MCP-error mapping.

Purpose: MCP-05 requires the SDK to be pinned and its package name + import paths verified against what is actually installed (the SDK has shifted between `@modelcontextprotocol/sdk` with `/server/mcp.js` + `/server/stdio.js` subpaths and newer package layouts — hardcoding from docs is how connect-time crashes happen). MCP-04 requires RelayError to map to an MCP tool-error result and `REDACTION_PATTERNS` to run before any value crosses the MCP boundary; centralizing that in result.ts keeps every tool wrapper honest and prevents a leaky path.

Schema note (MCP-03): the memory schemas this phase exposes are ALREADY exported from `src/contracts/memory.ts` (`recallSchema`/`RecallArgsSchema`, `rememberSchema`/`RememberArgsSchema`) — the single-source-of-truth requirement needs no new export work. The control-tool schema export that an earlier draft of this plan carried is DELETED: session-control tools are not exposed over MCP in v0.4 (killed scope — see 09-CONTEXT.md).

Output: pinned SDK + `resolveMcpSdk()`, and `src/mcp/result.ts` with `toMcpResult` / `relayErrorToMcpResult` / `withMcpResult`.
</objective>
