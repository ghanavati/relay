# Configuration

Relay reads config from three sources, in order of precedence (most specific wins):
1. CLI flags on each command
2. Environment variables
3. `~/.relay/config.json`

## Environment variables

### Core

| Var | Default | Purpose |
|---|---|---|
| `RELAY_DB_PATH` | `~/.relay/relay.db` | SQLite database path. Use `:memory:` for tests. |
| `RELAY_MEMORY_ALLOWED_WORKDIRS` | (unset) | Colon-separated allowlist of workdir prefixes that can write to memory. Unset = all allowed. |
| `RELAY_MAX_MEMORY_TOKENS` | `100000` | Token budget cap for memory store. `gcByTokenBudget` evicts non-pinned entries when exceeded. |
| `RELAY_RECALLED_LESSONS` | (unset) | Set to `1` to enable the recalled_lessons context layer for delegated tasks. |
| `RELAY_RUN_HISTORY_LAYERS` | (unset) | Set to `1` to inject last 5 successful runs as context for delegated tasks. |
| `RELAY_COLOR` | (unset) | `auto`, `always`, or `never`. CLI flag `--color=...` overrides. |

### Auto-extract pipeline

The auto-extract pipeline calls a local model on session-end to extract candidate memories from the transcript.

| Var | Default | Purpose |
|---|---|---|
| `RELAY_AUTO_EXTRACT_MODEL` | `qwen/qwen3-coder-next` | Model ID passed to the auto-extract endpoint. Should be a small, fast local model. |
| `RELAY_AUTO_EXTRACT_ENDPOINT` | `http://localhost:1234/v1/chat/completions` | OpenAI-compatible endpoint. Default = LM Studio. |
| `RELAY_AUTO_EXTRACT_TIMEOUT_MS` | `30000` | Hard timeout for the extraction call. On timeout, the pipeline writes `error:llm-timeout` to the log and exits 0 (non-fatal for the hook). |
| `RELAY_AUTO_EXTRACT_REQUIRE_BERRY` | (unset) | Set to `1` to require Berry MCP verification before any auto-extracted memory is written. With Berry unavailable, candidates are skipped, not written-as-unverified. |
| `RELAY_AUTO_EXTRACT_PROVIDER` | `lmstudio` | `lmstudio`, `openrouter`, or `anthropic`. Selects the API shape for the extract call. |
| `RELAY_AUTO_EXTRACT_ALLOW_REMOTE` | (unset) | Set to `1` to allow `RELAY_AUTO_EXTRACT_PROVIDER=openrouter` or `anthropic`. Default refuses non-local providers — protects against transcripts leaving the machine. |
| `RELAY_BERRY_ENDPOINT` | `http://localhost:7766` | Berry MCP HTTP endpoint for the optional hallucination check. |

### Provider keys

| Var | Default | Purpose |
|---|---|---|
| `OPENROUTER_API_KEY` | (unset) | Required for OpenRouter provider. Get from https://openrouter.ai. |
| `LMSTUDIO_ENDPOINT` | `http://localhost:1234` | LM Studio API endpoint for delegated runs. (Distinct from `RELAY_AUTO_EXTRACT_ENDPOINT`.) |
| `LMSTUDIO_API_KEY` | (unset) | Only needed if LM Studio is behind a reverse proxy with auth. |
| `ANTHROPIC_API_KEY` | (unset) | Required for `anthropic` provider. |
| `RELAY_CODEX_NETWORK_MODE` | (unset) | Set to `dangerous` to skip Codex approval flow (recommended for solo). |

## ~/.relay/config.json

Generated by `relay init`. Manual structure:

```json
{
  "providers": {
    "default": "codex",
    "codex": { "model": "gpt-5.3-codex", "reasoning_effort": "high" },
    "lmstudio": { "model": "zai-org/glm-4.7-flash" },
    "openrouter": { "model": "anthropic/claude-opus-4-5" }
  },
  "budget": {
    "openrouter_monthly_usd": 10
  },
  "memory": {
    "auto_pin_threshold": 3,
    "default_workdir": null
  }
}
```

## .relayignore

A per-workdir ignore file that controls what the auto-extract pipeline scrubs out of the transcript before sending it to the local model. Matches `.gitignore` glob syntax. Lines starting with `#` are comments. Place at the workdir root.

Example `.relayignore`:
```
# secrets
.env
.env.*
**/secrets/**
**/*.pem
**/*.key

# private notes
PRIVATE_NOTES.md
docs/internal/**

# generated artifacts (noise)
dist/**
node_modules/**
*.log
```

When the auto-extract pipeline reads the transcript, any tool-call result whose path matches a `.relayignore` rule is replaced with `[REDACTED:.relayignore]`. PII redaction (emails, API key shapes, tokens) runs after this on the surviving content. If redaction empties the transcript window, the pipeline exits with `skipped:empty-window`.

## .relay/auto-extract.json

Per-workdir consent file written by `relay memory auto-extract --enable [--workdir <path>]`. The SessionEnd hook refuses to run without this file.

Format:
```json
{
  "enabled": true,
  "consented_at": 1715000000000,
  "consented_by": "human",
  "workdir": "/Users/me/proj-a",
  "model": "qwen/qwen3-coder-next",
  "ttl_days": 30
}
```

Fields:
- `enabled` (boolean) — flip to `false` to disable without deleting the file
- `consented_at` (unix-ms) — when consent was granted
- `consented_by` (`"human"` or `"setup-script"`) — provenance
- `workdir` (absolute path) — must match the cwd at hook time, else hook skips
- `model` (string, optional) — overrides `RELAY_AUTO_EXTRACT_MODEL` for this workdir
- `ttl_days` (integer, optional, default `30`) — TTL for memories written by this workdir's auto-extract

`relay memory auto-extract --disable` flips `enabled` to `false`. To revoke completely, delete the file.

## Database location

Relay uses one SQLite file. Default location resolves in order:
1. `$RELAY_DB_PATH` if set
2. `~/.relay/relay.db` (creates parent dir on first write)

The DB is single-writer. Don't run two `relay` processes against the same DB simultaneously.

## Per-project workdir scoping

Memory entries can be scoped to a workdir. Pass `--workdir /path/to/project` to recall/remember commands. Unscoped entries are global.

For multi-project setups, use the `RELAY_MEMORY_ALLOWED_WORKDIRS` allowlist:
```
export RELAY_MEMORY_ALLOWED_WORKDIRS=/Users/me/proj-a:/Users/me/proj-b
```
When this is set, recall and show-context default `--workdir` to the current cwd.

## Logging

Relay v0.2.0 writes a centralized append-only NDJSON log at `~/.relay/relay.ndjson`. One JSON object per line. Used by `relay memory tail` and `relay info`. Rotates at 10MB or 30 days, whichever first.

The auto-extract hook also appends a separate one-line log per run to `~/.relay/auto-extract.log` (status: `ok`, `skipped:no-consent`, `skipped:empty-window`, `error:llm-down`, `error:parse`, `error:berry-failed`).

Pipe individual commands to your own files if you want command-specific history:
```
relay memory recall test --json | tee -a ~/.relay/recall.log
```

## Pause sentinel

`~/.relay/paused` blocks the SessionStart and SessionEnd hooks. Created by `relay pause`, removed by `relay resume`. The hooks check the sentinel before doing any work and exit 0 if present.

## Secrets

Provider API keys: env vars only. Add to your shell rc file:
```bash
export OPENROUTER_API_KEY=<your-openrouter-key>
export ANTHROPIC_API_KEY=<your-anthropic-key>
```

A `~/.relay/secrets` keystore is planned for v0.3.

## Configuration precedence example

For `relay memory recall test --type lesson`:
1. `--type lesson` from CLI overrides config defaults
2. `RELAY_MEMORY_DEFAULT_TYPE=fact` env var would have been used if no flag
3. `~/.relay/config.json` `memory.default_type` would have been used if neither
