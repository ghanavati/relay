# v0.1.0 deep retro — how I'd do it differently

Written 2026-05-02 after v0.1.0 ship. Inputs: 4 parallel CC subagent audits (bloat / docs / CLI-vs-best-in-class / cmd-* surface) + 3 LM Studio dispatches (LFM2, Qwen3-Coder-Next, GLM-4.7-Flash) + Codex post-ship verification.

## What shipped

Solo CLI extracted from relay-mcp. ~5KB AGENTS.md, 1463 lines across 8 cmd files + cli.ts, 360/360 tests stable, AGPL-3.0, private repo. Surface: `run`, `parallel`, `history`, `diff`, `compare`, `doctor`, `init`, `memory {remember,recall,show-context,get,hook,to-rules}`. Workers: codex, openrouter, lmstudio, anthropic.

## What worked

- **Whitelist + fresh-write extraction.** Switching from rsync-exclude to "list every file we want, fresh-write all orchestration" cut tsc errors from 130 → 0 in 2 iterations.
- **Codex as ship-gate.** Caught 4 ship-blockers (unwired `memory show-context`, stale `--help`, dead `cmd-corpus.ts`, stale CHANGELOG) that I missed in self-review. Worth every second.
- **`--test-concurrency=1` + `beforeEach DELETE`.** 7/7 stability runs. The `:memory:` SQLite singleton + parallel test runner is a known foot-gun; this combination kills it dead.
- **Slim DB path (`~/.relay/relay.db`).** Ditching the `~/.relay-mcp/` legacy path means the slim CLI doesn't inherit the parent project's bloated audit trail.

## What broke (specific to this session)

1. **LFM2 used for the wrong job.** I asked LFM2 to "read these two doc files and produce a 1-page compressed retro." It returned an `echo "..."` shell stub with placeholder text. Liquid AI's published guidance: LFM2 is for agentic tool-use / summarization-of-text-in-prompt / RAG generation / customer support — **NOT for "read files and produce X."** Correct LFM2 usage: paste the source content directly into the prompt, then ask for compression. Don't expect it to do file I/O reliably.
2. **GLM-4.7-Flash bloat-pattern in mechanical work.** Asked for a v0.2 plan table; got 100 rows where ~60% were filler ("cmd-run.ts add task param: keep S already implemented" × 28). Real signal in 40 rows. GLM template-pads to perceived expectations. Fix: cap row count in the prompt ("MAX 30 rows. Quality > coverage.").
3. **CC general-purpose subagent context-leak.** The bloat-audit agent cited `src/runtime/continuity/recipe.ts`, `src/tools/handoff.ts`, `src/cli/cmd-corpus.ts` — none exist in this slim Relay repo. All three exist in the parent `relay-mcp`. Without an explicit "this is NOT relay-mcp" framing, the agent's familiarity bled through. Fix: every subagent prompt for this repo must start with `IMPORTANT: this is the slim solo-CLI Relay distribution at /Users/ghanavati/ai-stack/Projects/Relay. It is NOT the parent /Users/ghanavati/ai-stack/Projects/relay-mcp — those are different codebases.`
4. **Qwen3-Coder-Next dispatch rejected by trust gate.** "Worker lmstudio:qwen/qwen3-coder-next does not meet trust requirements for standard-risk task." I should have known this from the relay-mcp dispatch policy — model needs explicit trust registration before dispatch. Wasted one slot in the parallel batch.
5. **Stale doc artifacts shipped.** `docs/parallel.md:3` said "parallel dispatch arrives in v0.2" after `relay parallel` shipped. `SECURITY.md:73` said "when implemented in v0.2." `docs/recipes/morning-startup.md:1-23` had worker-prompt boilerplate accidentally committed by the worker that wrote it. `AGENTS.md:115` referenced `docs/provenance.md` that doesn't exist. **All caught by the doc-redundancy audit, all fixed in this same retro session.** Lesson: every worker that writes a doc file must have its output stripped of `<!-- layer:worker_constraints -->` headers and trailing commit boilerplate by the orchestrator before merge.
6. **`cmd-init.ts:36` hardcoded MY machine path.** `CC_MEMORY_PATH_CANDIDATE` resolves only on `-Users-ghanavati-ai-stack-Projects-relay-mcp`. Will never trigger for any other user. Should derive from `$HOME/.claude/projects/<project-hash>/memory/` dynamically.
7. **`cmd-doctor.ts:92` lies on failure.** Hardcoded "All checks passed." even when the summary shows failures. Subtle bug that escaped both Codex and human review.

## What I'd do differently for v0.2

In execution-priority order:

1. **Cap LFM2 to its publisher-stated use cases.** Routing classification (with a known label set), summarization-with-text-pasted-in, RAG generation. Never "read this file." Never code. If you want a doc compressed, paste the doc text in the prompt.
2. **Boilerplate every subagent prompt with the repo identity disambiguation.** Two-line preamble: which repo, which is NOT this repo, where the source of truth lives.
3. **Strip worker-prompt artifacts before merge.** Add a 5-line pre-merge filter: drop any `<!-- layer:worker_constraints` line, drop any trailing `git add ... && git commit -m '...'` line. The relay parallel worker should do this in the merge step.
4. **GLM prompts: cap output size explicitly.** "MAX 30 rows. Drop filler. Quality > coverage." Otherwise GLM template-pads.
5. **Fix the 3 latent bugs** caught by the cmd-* audit: `cmd-doctor.ts:92` All-passed lie, `cmd-init.ts:36` hardcoded path, `cmd-memory-ops.ts:180` to-rules duplicate-append.
6. **CLI ergonomics pass.** Adopt 5 things from ripgrep/fd/gh: NO_COLOR + auto color detection, `relay completion bash|zsh|fish`, short-flag aliases (`-p`, `-w`, `-m`), context-sensitive `--help`, NDJSON for `--json` (one object per line) instead of wrapped object. Estimated cost: ~600 lines, no architectural change.
7. **Merge `cmd-diff` into `cmd-compare`.** `compare <a> <b>` two-arg = side-by-side; `compare <a>` one-arg = current diff-show behavior. Saves one entrypoint, ~60 lines.
8. **Extract `src/cli/probes.ts`.** Doctor + init both probe codex/lmstudio/anthropic/openrouter independently. ~50 lines duplicated.
9. **Defer `relay budget` properly.** Currently flagged "future stub" with `exit 64` but listed in help text and CHANGELOG. Either ship `--scope=provider` redesign in v0.2 or remove from public help.
10. **`relay history` should default to `--workdir .`** when run inside a git repo. Most users want "what did I run *here*."

## What would make v1.0 superior to v0.1.0

- **Single-binary distribution.** `npm install -g relay` works but bun+pkg → one ~50MB binary that runs without Node. Solo users never want to manage Node versions.
- **Zero-config first run.** `relay init --quick` already exists; make it the default when no `~/.relay/` directory exists. Users should never have to read configuration.md before their first task succeeds.
- **`relay watch <dir>` for continuous dispatch.** Drop a JSON spec into a watched dir → runs it. Delete the file when done. The simplest possible queue. No daemon.
- **Brew formula.** `brew install ghanavati/tap/relay`. Solo CLIs without brew never feel native.
- **Two-tier worker selection: explicit + auto.** `relay run "task"` (no provider) → router picks GLM-flash for ≤200 chars / Codex for ≥800 chars / LM Studio default model otherwise. Override with explicit `--provider`. Today every command requires a provider choice.

## What I'd cut (still in v0.1.0)

- `cmd-init.ts:36-39` `CC_MEMORY_PATH_CANDIDATE` hardcoded path — replace with dynamic discovery.
- `cli.ts:262` `install || !uninstall` default-to-install logic for `memory hook` — surprising; require an explicit verb.
- `cli.ts:350` `futureCmds = ['corpus', 'budget']` — either ship them in v0.2 or stop pretending they're "coming." Today they're 64-exit dead surface.
- All `<!-- layer:worker_constraints hash:... -->` comments in any committed file — they're pure worker-prompt pollution.

## Honest verdict

v0.1.0 is **competent intermediate** as a solo CLI: bones are right, ergonomics 6/10. The hard parts (delegation engine, audit trail, FTS5 memory, parallel dispatch with bounded concurrency) all shipped and are stable. What's missing is 600 lines of CLI polish (colors, completion, short flags, NDJSON, smart defaults) that costs nothing architecturally but separates "internal tool" from "feels native."

The biggest *process* lesson: **subagents and LM Studio workers each have a narrow use case.** LFM2 for code = wrong tool. GLM for unbounded prose = padding. CC general-purpose without repo-disambiguation = context leak from training. Knowing which model + which framing for which task is the actual leverage. Routed wrong, you waste a parallel slot AND get hallucinated output.

## Subagent routing matrix (validated this session)

| Task | Right tool | Wrong tool (and why) |
|------|------------|----------------------|
| Cross-file code review | CC subagent (general-purpose or feature-dev:code-explorer) — but **only with explicit "this is repo X, NOT repo Y"** preamble | LFM2 (won't reason across files); LM Studio anything (no inter-file graph) |
| Mechanical/structured table generation | GLM-4.7-Flash with **MAX rows cap** | LFM2 (returns shell stubs); CC subagent (overkill) |
| Summarization of text-in-prompt | LFM2 with full source pasted inline | LFM2 with "read this file" instruction (will return placeholder); GLM (template-pads) |
| Routing classification with known labels | LFM2 (50% accuracy in peer testing — provisional, verify per-batch) | CC subagent (overkill); GLM (will rationalize) |
| Code generation, single file ≤100 lines | GLM-4.7-Flash, prompt with full API surface inline | LFM2 (publisher says NOT for code); CC general-purpose (token-expensive vs LM Studio cost-free) |
| Code generation, cross-file or > 100 lines | CC inline, no delegation | LM Studio anything (one-shot ceiling) |
| Frontier reasoning critique | Codex CLI or OpenRouter Claude | Local models (insufficient depth) |

This matrix is the actionable retro: pick by task shape, not by what's available.
